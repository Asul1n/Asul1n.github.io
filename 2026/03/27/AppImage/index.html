AppImage

发表于2026-03-27|更新于2026-03-28|Linux
|浏览量:

在 Linux 中，传统的安装方式（如 apt install）依赖于系统的共享库。如果你的系统库版本太旧或太新，软件就跑不起来。

一，Write Once, Run Everywhere

AppImage 解决了这个问题： 它把程序运行所需的所有依赖库、图标、资源文件全打包进了一个压缩包里。当你运行它时，它会挂载一个虚拟文件系统，直接从这个包里读取依赖，而不依赖你的 Ubuntu 系统库。

当你双击一个 Ao.AppImage 时，后台发生了以下过程：

  1. 挂载 (Mounting): AppImage 文件包含一个 FUSE（Filesystem in Userspace）映像。它会被挂载到一个临时目录下（通常是 /tmp/.mount_XXXX）。

  2. 执行 (Execution): 系统运行该目录下的 AppRun 脚本。

  3. 自包含运行: 程序开始运行，它优先加载包内自带的库，而不是 /usr/lib 里的。

  4. 卸载 (Unmounting): 当你关闭程序时，临时挂载点会自动消失，不留任何痕迹。

第三步：固定到启动栏（集成到系统）

AppImage 最大的缺点是默认不会出现在你的应用菜单（Gnome Launcher）里。

  1. 安装集成工具（最推荐）： 安装 AppImageLauncher。它会自动监测你运行的 AppImage，并问你是否要“集成到系统”。集成后，它会出现在应用列表里，你就能把它固定到 Dock 栏了。
