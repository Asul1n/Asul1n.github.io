Latex in Hexo（butterfly + pandoc 配置）

发表于2026-03-15|更新于2026-03-15|hexo
|浏览量:

在使用 Hexo 搭建博客时，默认的渲染引擎（如 marked 或 kramed）经常会误解析 LaTeX 中的特殊符号（如 _, \\, -），导致公式渲染失败。本文记录如何通过 Pandoc 引擎实现与 Obsidian 一致的数学公式体验。

1. 环境准备

首先，确保你的系统中已安装 Pandoc 软件。

  • Linux (Ubuntu/Debian): sudo apt install pandoc

  • macOS: brew install pandoc

  • Windows: 前往 Pandoc 官网 下载安装包。

5. Obsidian 写作注意事项

为了确保在 Hexo 网页端完美显示，建议在 Obsidian 写作时遵循以下习惯：

  • Front-matter 声明：在笔记开头必须包含 mathjax: true。

  • 行内公式空格：建议在行内公式 $公式$ 的左右两侧各加一个空格，例如：这是 $p-\frac{1}{2}$ 公式。

  • 块级公式空行：$$ 包裹的块级公式，上下各留一行空行
