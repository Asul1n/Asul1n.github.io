浮点数的规格化

发表于2026-03-26|更新于2026-03-27|计组
|浏览量:

在计算机组成原理中，规格化（Normalization） 是浮点数运算中至关重要的一步。它的核心目的是：在保持数值不变的前提下，通过调整阶码和尾数，使尾数的绝对值尽可能大，从而最大程度地利用尾数的有效位数，减少舍入误差。

1. 规格化的标准形式

对于一个补码表示的浮点数，规格化要求其尾数 M 的绝对值必须落在 \([\frac{1}{2},1)\) 或 \((−1,−\frac{1}{2}]\) 之间。

判断是否规格化的“金标准”是观察符号位（S）与数值最高位（Ms​）：

  • 规格化要求：符号位与数值最高位必须相反。
数值类型 补码形式 (单符号位) 含义
正数 \(0.1 \phi \phi \dots \phi\) 范围在 \([0.5, 1)\)
负数 \(1.0 \phi \phi \dots \phi\) 范围在 \((-1, -0.5]\)

  • 如果是 0.0…，说明有效数字在后面，白白朗费了前面的位数。

  • 如果是 1.1…，在补码中这代表一个绝对值很小的负数（如 −0.0001），同样浪费了精度。

2. 左规（Left Normalization）

当运算结果的尾数不是规格化数（即出现 0.0… 或 1.1…）时，需要进行左规。

  • 操作： 尾数每左移一位，阶码减 1。

  • 停止条件： 直到符号位与数值最高位不同（即变成 0.1… 或 1.0…）。

  • 注意： 左规可能会进行多次。

例子： 设结果为 \(2^5×(0.0011…)\)

  1. 左移一位：\(2^4×(0.011…)\)

  2. 再左移一位：\(2^3×(0.11…)\) —— 完成规格化！

3. 右规（Right Normalization）

当运算结果的尾数溢出（即出现 01.… 或 10.…）时，需要进行右规。

  • 操作： 尾数右移一位，阶码加 1。

  • 停止条件： 右规只需进行一次，因为双符号位溢出后，右移一位必然回到规格化范围。

  • 低位处理： 右移出的位通常需要参与“舍入”处理。

例子： 设结果为 \(2^3×(01.10…)\)（正溢出）

  1. 右移一位：\(2^4×(0.011…)\) —— 错误！

  2. 正确右移：符号位变回 00，结果为 \(2^4×(0.110…)\) —— 完成规格化！

4. 双符号位补码规格化判别准则

  • 双符号位（变形补码）：使用两位符号位 \(M_{S_1}​M_{S_2}\)​。

    • 00：正数

    • 11：负数

    • 01 / 10：发生溢出（正溢或负溢）

  • 规格化定义：为了提高运算精度，要求尾数的绝对值在 \([0.5,1)\) 之间。对于补码而言，规格化的特征是符号位与数值位最高位（\(M_1\)​）相反。

我们将结果形式记为：\(M_{S_1}​M_{S_2}​.M_1​M_2​…M_n\)

状态 符号与 \(M_1\)​ 组合 典型示例 处理动作 原因
不需要规格化 00.1… 或 11.0… \(00.1011\) / \(11.0101\) 无 已经是规格化数，符合 \(S \oplus M_1 = 1\)
需要左规 00.0… 或 11.1… \(00.0011\) / \(11.1101\) 尾数左移，阶码减 1 数值绝对值太小，有效位隐藏在后面
需要右规 01…. 或 10…. \(01.1011\) / \(10.0101\) 尾数右移，阶码加 1 运算结果溢出，需将溢出位移回数值位

5. 特殊情况：补码 -0.5 和 -1

在补码规格化中，有两个特殊的边界点需要注意：

  1. −1/2 的补码：1.100…。这不是规格化数，因为它不满足“符号位与数值位相反”，必须左规变成 1.000…。

  2. −1 的补码：1.000…。这是规格化数。虽然它的数值位全是 0，但它满足符号位（1）与数值最高位（0）相反。
