Vector

发表于2026-04-16|更新于2026-04-16|cpp
|浏览量:

vector 在内存中是连续存储的。这意味着它支持快速的随机访问，但在中间插入或删除元素的效率较低。

  • Size (大小)：当前容器中实际存储的元素个数。

  • Capacity (容量)：容器在不重新分配内存的情况下最多能容纳的元素个数。

  • 扩容机制：当 size 超过 capacity 时，vector 会申请一块更大的内存（通常是原容量的 1.5 或 2 倍），将原数据拷贝过去，然后释放旧内存。

添加与删除

  • push_back(val)：在末尾添加元素。

  • pop_back()：删除末尾元素。

  • insert(pos, val)：在指定迭代器位置插入元素（涉及数据移动，较慢）。

  • erase(pos)：删除指定位置的元素。

访问元素

  • v[i]：下标访问，速度最快但不检查越界。

  • v.at(i)：会检查越界，如果越界会抛出异常，更安全。

  • v.front() / v.back()：访问首尾元素。

函数 功能 复杂度
size() 返回元素个数 \(O(1)\)
empty() 判断是否为空 \(O(1)\)
clear() 清空所有元素 \(O(n)\)
begin() / end() 返回首尾迭代器 \(O(1)\)
shrink_to_fit() 释放多余的 capacity 空间 \(O(n)\)

隐患一：隐式类型转换 (Type Mismatch)

nums.size() 返回的是 size_t（通常是 unsigned long long），而 i 是 int。

  • 安全性：在 64 位系统下，当数据量超过 \(2^{31}−1\) 时，int 会溢出。

  • 性能开销：无符号数与有符号数比较时，编译器会进行隐式类型转换。在底层指令中，这可能增加符号位扩展的操作。

隐患二：重复调用带来的开销

虽然现代编译器能优化掉简单的 size() 调用，但如果循环体很复杂，编译器可能无法确定 nums.size() 是否会改变，从而在每次迭代时都重新读取内存中的长度值。
